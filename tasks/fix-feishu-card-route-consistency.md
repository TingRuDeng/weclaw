# 飞书卡片路由一致性修复

## 目标

- 保留结构化问答卡片的关联键，避免并发问题串线。
- 让 `/mode` 按平台会话路由隔离。
- 阻止模型旧卡片操作与出卡时不同的 Agent。
- 补齐跨层回归测试。

## 非目标

- 不修改用户可见命令名称。
- 不使用 `conv` 或 ChatID 推测完整飞书会话。
- 不兼容缺少必要元数据的旧模型卡片。

## 当前事实

- 飞书按钮依赖 `feishu_session_key` 恢复消息路由。
- 普通卡片回调当前丢弃结构化问答的 `approval_key`。
- `/mode` 当前按真实用户 ID 存储，与窗口会话不一致。
- 模型卡片当前未记录出卡时的 Agent。

## 决策日志

- 2026-07-13：普通回调只透传已有的关联字段，不把结构化问答误分类为权限审批。
- 2026-07-13：审批模式统一使用 `routeUserID`，无独立路由的平台自然回退到用户 ID。
- 2026-07-13：模型卡片记录目标 Agent；缺失或不一致时明确失效，不静默回退。
