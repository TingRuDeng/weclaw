# 当前任务记录

## 目标

阻止飞书延迟投递的历史普通消息进入 WeClaw 命令路由或 Agent 会话，避免旧 `/restart`、`/cc ls`、`/cx ls` 和普通文本在服务恢复后被误执行。

## 非目标

- 不调整微信消息处理。
- 不调整飞书卡片点击回调。
- 不调整 Codex、Claude 会话路由与回复内容。
- 不要求现有配置手工增加新字段。

## 当前事实

- 飞书延迟消息以新的消息 ID 首次到达，现有 `message_id` 去重无法识别。
- 远端消息创建时间比 WeClaw 实际接收时间早约 65 分钟，并且早于本次服务启动时间。
- 飞书消息当前在 `feishu.Adapter.handleMessageEvent` 中直接进入解析与分发，没有创建时间水位校验。

## 决策日志

- 使用“本次运行启动水位 + 最大消息年龄”双重保护。
- 默认最大消息年龄为 120 秒；每个飞书机器人可通过 `max_message_age_seconds` 覆盖，0 表示关闭，负数为非法配置。
- 过期或时间戳非法的消息只记录结构化日志，不向飞书回复。

## Review 小结

Review Gate 终态为 `finished`，结论通过。飞书普通消息在解析、资源下载、命令路由和 Agent 分发前执行启动水位与最大年龄校验；默认窗口为 120 秒，0 可显式关闭，负数和无法安全转换为 `time.Duration` 的超大值会被配置校验拒绝。过期日志包含机器人、会话、消息 ID、创建时间、接收时间、延迟和原因，不包含消息正文。启动前消息、运行期超龄消息、非法时间戳、正常消息、关闭保护、配置保留和重启提示均有回归测试；新核心文件函数覆盖率为 100%。全仓测试、目标包 Race、Vet、Staticcheck、构建、文档契约和差异检查全部通过。剩余风险是本机时钟严重偏移或真实投递延迟超过窗口时消息会被丢弃，用户需重发或显式配置 0 关闭保护。飞书卡片点击事件按已确认非目标保持原行为。
