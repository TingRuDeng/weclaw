# WeClaw 可靠性加固任务

## 目标

按用户确认的顺序完成以下修复：

1. 普通消息等待前序操作时提供一次明确反馈，同时保持严格顺序。
2. 统一任务停止状态转换，消除停止与自然终态竞态。
3. 规范化工作空间真实路径，并补齐配置加载安全契约。
4. 为飞书消息去重处理权增加内存租约续租。
5. 抽取 Codex、Claude ACP 和同步 Agent 的公共任务生命周期。

## 非目标

- 不改变命令名称、会话键和持久化格式。
- 不改变 Codex owner、Desktop、rollout 和中断核对语义。
- 不让普通消息越过会话切换或前序消息。
- 不持久化飞书 `processing` 去重状态。
- 不进行与任务生命周期无关的 Handler 大规模拆分。

## 当前事实

- 当前 `HEAD` 与 `origin/main` 一致，基线提交为 `339e9f5`。
- 工作区包含上一轮尚未提交的审查修复，本任务必须在其上增量修改。
- 普通消息当前无限等待前序票据，只有 `/stop` 可以超时后继续。
- 停止状态更新分散在本地取消和外部 Codex 中断路径。
- `/cwd` 白名单会解析符号链接，但实际传给 Agent 的路径仍可能是符号链接路径。
- 飞书去重 `processing` claim 固定 10 分钟，没有续租。
- Codex、Claude ACP 和同步 Agent 分别维护任务准入、进度和终态流程。

## 决策日志

- 排队超过 3 秒后只提示一次，提示不进入同一顺序队列。
- 普通消息永不因等待超时而乱序执行；`/stop` 保留控制命令旁路。
- 停止状态由 `activeAgentTask` 原子转换并返回明确结果，取消动作在锁外执行。
- `/cwd` 在输入边界解析真实路径，后续校验、保存和执行复用同一路径。
- 去重处理权采用仅内存的 owner 租约续租，不以延长 TTL 或持久化 processing 替代。
- Agent 公共协调器只负责生命周期，协议和会话解析仍由各 Agent 策略负责。
- 同步 Agent 接入单条排队，但底层 Agent 调用仍保持同步语义。
